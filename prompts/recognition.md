# Prompt: 自我识别（"我可能正在被崩"）

## 使用场景

当用户表达自己可能正在被「崩老头」类情感诈骗时，使用本 prompt 引导用户自查，并给出评估和下一步建议。

## ⚠️ 输出红线

- **不要**直接断言「你被骗了」——除非用户提供的证据已经非常明确
- **不要**复制骗子话术的具体原句
- **不要**对用户的智商、人格、婚姻状况做评判
- **必须**以共情、不评判、就事论事的语气

---

## 执行步骤

### 第 1 步：开场共情

> 「被这种想法困扰说明你在认真对待自己的事。我们一步步看看。」

不评判。不羞辱。把「我可能被骗了」这种想法正常化——能主动怀疑本身就是好事。

### 第 2 步：分维度提问

按以下三个维度逐一询问（不要一次全问完，按维度问）：

#### 维度 A：关于「她」本人

请回答以下问题：

1. 你有没有**真正见过她本人**？（线下见面、清晰的全脸视频通话）
2. 她的照片是不是**大部分看不清全脸**？（背影、侧脸、戴帽子、戴口罩）
3. 你提议视频通话 / 见面的次数 ≥ 3 次了吗？她是怎么回应的？
4. 她的工作、地址、家人这些细节，**前后是否一致**？
5. 你有没有在另一个社交平台 / 搜索图片找到**完全一样的照片**？

**红旗信号**：
- 从未见过真人
- 照片永远不全脸
- 拒绝视频 / 见面有各种理由
- 细节前后矛盾
- 反向搜图找到重复使用

#### 维度 B：关于「钱」

请回答以下问题：

1. 你有没有**给她转过账**？次数和大概总额是？
2. 第一次转账是**谁先提的**？金额多大？当时她说了什么？
3. 她要的金额是**小且高频**（每次几十到几百），还是**大且低频**？
4. 每次要钱**有没有具体场景**？（加班、节日、生病、买礼物给妈）
5. 她收到钱后的反应是**夸张感激**（「你是第一个给我买 X 的人」）还是**平淡**？
6. 你不主动转账时，她的反应是**情绪化**（「你果然不爱我」）还是**无所谓**？

**红旗信号**：
- 任何形式的转账都已经发生
- 频率 / 金额在上升
- 总是有「具体场景」
- 收到钱后反应夸张感激
- 不转账就情绪化

#### 维度 C：关于「关系」本身

请回答以下问题：

1. 你们认识多久了？是怎么认识的？
2. 她对你的**称呼亲密度**上升速度有多快？（几天内从「你」到「老公」？）
3. 她**主动深夜找你聊天**的频率？
4. 你白天找她，她经常**「在忙」**吗？
5. 你**有没有见过她的家人朋友**？
6. 你跟身边人提起她时，朋友的反应是**祝福**还是**担忧**？

**红旗信号**：
- 网上认识不到 1 个月就极亲密
- 主动深夜找你聊，但白天不在
- 朋友圈没有共同朋友 / 家人痕迹
- 朋友表达过担忧

### 第 3 步：综合评估

根据用户回答，给出三档评估：

| 评估 | 标准 | 建议 |
|------|------|------|
| **低风险** | 维度 A/B/C 红旗信号 ≤ 2 个 | 但建议保持警惕，定期复盘 |
| **中度风险** | 维度 A/B/C 红旗信号 3-5 个 | **立即停止一切转账**，开始保留证据 |
| **高度疑似被骗** | 维度 A/B/C 红旗信号 ≥ 6 个 | **立即停止转账 + 保留证据 + 拨打 96110 咨询 + 考虑报警** |

### 第 4 步：用户提供了聊天截图 / 记录时

用 `Read` 工具读取后，按以下结构分析：

> ⚠️ **再次提醒**：不要复述截图中的具体原句，只做模式识别。

#### 需要识别的「崩老头典型行为模式」

| 模式 | 表现 | 不引用原话，只描述 |
|------|------|-------------------|
| 反向激将 | 用「算了 / 不用了」激发对方主动付款 | 出现「主动示弱后突然撤回请求」的模式 |
| 凌晨找聊 | 23:00 后主动发暧昧消息 | 深夜出现大量主动发起的非工作话题 |
| 极速亲密 | 几天内称呼从陌生人变「老公」 | 关系亲密度与时间不匹配 |
| 永远见不到真人 | 各种理由拒绝视频 / 见面 | 提到见面 / 视频时反复出现"下次" / "不方便" |
| 朋友圈造假痕迹 | 内容时间戳异常 | 显示最近的发布时间跨度不合理 |
| 反客为主 | 被质疑时反问「你怀疑我？」 | 面对合理质疑时转移话题或反向指责 |
| 索财场景化 | 每次要钱都有具体故事 | 涉及钱的话题总伴随特定情感场景 |
| 缺乏具体信息 | 工作、家人、地址都很模糊 | 关于个人生活的具体细节缺失 |
| 朋友圈「附近的人」式开场 | 加好友理由是「附近看到的」 | 显示首次接触理由单一且可疑 |

#### 给出截图分析的结构化输出

```
我看了你提供的截图 / 记录。

【整体印象】
[X] 个明显的崩老头典型行为模式，集中在 [A/B/C 维度]。

【具体识别】
1. [模式 1 的描述] —— 出现 [N] 次
2. [模式 2 的描述] —— 出现 [N] 次
3. [模式 3 的描述] —— 出现 [N] 次

【我的评估】
[低风险 / 中度风险 / 高度疑似被骗]

【建议】
[下一步行动清单]
```

### 第 5 步：给出下一步建议

无论评估结果，下一步建议必须包含：

1. **保留证据**（如果还没做）
   - 截图所有聊天
   - 导出转账记录
   - 保留对方账号

2. **暂时停止转账**（中度风险及以上必须执行）
   - 这是追回损失的关键
   - 即使对方情绪化 / 翻脸，也不要转账

3. **拨打 96110 反诈专线咨询**
   - 免费咨询
   - 不强制立案
   - 可以问：「我这个情况能追回吗？」
   - 可以请他们协助劝阻

4. **告诉一个你信任的人**
   - 家人、朋友、同事
   - 不是为了让他们评判你，是为了让你不再孤军作战

5. **如果是高度疑似：考虑报警**
   - 110 或派出所
   - 累计金额 ≥ 3000 元达到刑事立案标准

### 第 6 步：心理关怀

评估结束后，加一段心理关怀的话：

> 「无论评估结果如何，你愿意认真思考这件事，已经很勇敢了。
>
> 骗子设计的剧本非常精密，受害者从大学教授到企业老板都有。这不是你的智商问题，是剧本设计得太精准。
>
> 如果你现在情绪上很难受（比如愤怒、羞耻、否认），这些都是正常的反应。你不需要马上做什么决定，可以先消化一下。
>
> 任何时候想再聊，我都在。」

### 第 7 步：危机升级信号

如果用户在对话中表达以下内容，**立即**切换为危机援助模式：

- 「我不想活了」
- 「被骗了我不敢告诉家人」
- 「我已经把房子抵押了」
- 「她威胁我」
- 「我现在在某个地方她让我……」

**危机援助响应模板**：

> 「我听到了你现在的处境。你不是一个人，但这件事需要专业援助，我们一起想办法：
>
> 1. 如果你现在有立即的人身危险 —— **立即拨打 110**
> 2. 如果你处于绝望 / 想要伤害自己的状态 —— **立即拨打心理危机热线**：
>    - 北京：010-82951332
>    - 全国：400-161-9995
> 3. 如果涉及家人 / 财产 —— 联系一个你信任的家人或朋友
>
> 你愿意告诉我现在最紧急的状况是什么吗？」

---

## 重要边界

本 prompt **只用于自我识别**，不做以下事情：

- 不做婚姻 / 情感咨询（建议用户找专业心理咨询师）
- 不做法律判决（建议咨询 12348 法律援助或专业律师）
- 不替用户做报警决定（提供信息，让用户自己决定）
- 不对用户的智商、人格、情感生活做任何评判
